# PRD Questionnaire Template

## SECTION 1 — Vision & Target Users
1. What are the primary use cases?
2. Who is the target user today?
3. Should the system behave like a strict automation or collaborative copilot?
4. What is the success metric for Phase 1?
5. Long-term system feel: product, experiment, or portfolio showcase?

## SECTION 2 — Resume Inputs & Output Requirements
1. Source of canonical resume?
2. Number of templates?
3. Formatting constraints?
4. Elements adjustable per job?
5. Output formats needed.

## SECTION 3 — Cover Letter Logic
1. Template strategy.
2. Word count constraints.
3. Must-include items.
4. Tone preferences.
5. Automatic project pulling?

## SECTION 4 — Critic Agent & Rubric
1. Critique targets.
2. Strictness level.
3. Rubric items.
4. Refinement strategy.
5. Revision history needs.

## SECTION 5 — Company Intelligence & Networking
1. Automatic scraping?
2. LinkedIn integration model.
3. Contact ranking criteria.
4. Outreach drafting automation.
5. Output formats.

## SECTION 6 — Architecture Preferences
1. Languages.
2. Storage options.
3. LLM choices.
4. Runtime environment.
5. UI vs CLI.

## SECTION 7 — Future Phases
1. Job tracking?
2. Interview prep?
3. Profile variants?
4. Monetization?
5. Positioning: open-source, proprietary, portfolio, commercial?
